会员电商系统抽奖活动 - DANBAOLAI文档

一、功能介绍

抽奖活动凭借着以小博大的杠杆效应、低门槛参与、高奖励诱惑的活动机制，无论是线下门店促销，还是线上活动，都被广泛用于拉新、促活、获客等增长环节。
我们采用九宫格/大转盘抽奖的方式，后台可设置8个奖品，可自由调节奖品的中奖概率；在关注公众号、商城的多个环节、以及积分消耗等多处，均可设置抽奖活动

二、操作流程

营销 ＞ 抽奖列表

三、功能说明

1. 订单评价抽奖

a、用户评价商品后，可于评价成功页面参与抽奖
b、一个订单有多个商品时，每次评价都可抽奖
抽奖次数：设置一次成功下单后获得的抽奖次数。

移动端展示

2. 下单支付抽奖

用户成功下单后，可于支付成功页面参与抽奖
抽奖次数：设置一次成功下单后获得的抽奖次数。

移动端展示

3. 关注公众号抽奖

a、用户关注商城公众号后，可参与抽奖
b、用户第一次关注公众号才会获得抽奖资格
c、用户可通过邀请好友关注公众号增加抽奖次数
抽奖次数：设置成功关注公众号后获得的抽奖次数
邀请新用户最多可获得抽奖：通过邀请好友最多可获得的抽奖次数
邀请一位新用户关注公众号可获得抽奖：邀请一位好友成功关注公众号后获得的抽奖次数

4. 积分抽奖

用户可通过消耗积分来抽奖,系统自动计算用户的积分可抽奖次数
用户可按需选择抽奖的样式（九宫格/大转盘）

活动名称：可添加抽奖活动名称
活动时间：设置本次抽奖活动的开始和结束时间，用户在设定时间内可参与活动抽奖。
抽奖类型：仅积分抽奖有类型的区分。抽奖类型分为大转盘抽奖和九宫格抽奖，两种抽奖类型的移动端样式不同，区别如下图，商家可按需设置。
参与用户：

全部用户：系统所有用户均可参与本次抽奖活动
部分用户：可选择用户等级，用户类型，用户标签以区分可参与抽奖活动的用户。


每人抽奖总次数：：每个账户可参与本次抽奖活动的总次数
每人每天抽奖次数：每个账户每天可参与本次抽奖活动的总次数
单次抽奖消耗积分：用户参与本次积分抽奖活动时，抽奖一次消耗的积分数量。
规则选择：

优惠券：此处选择的优惠券为后台赠送类型的优惠券。权重指中该奖的概率，权重越大，中奖率越高；奖品名称和奖品图片于抽奖页面显示；奖品数量指抽奖活动中该奖品的总数量；提示语指抽中某一奖品时的提示信息。
积分：需设置积分数量。中奖后自动发放到用户的积分账户
商品：选择中奖商品时只能选择其中一种规格，需要在奖品名称或活动规则中说明。
红包：需维护中奖金额。用户领取红包后会自动到账微信零钱，添加此奖品需开通微信支付,并且账户中金额不能小于0.01元；需开通商户自动到零钱功能才可使用。
余额：中奖后自动发放到用户余额账户。需维护中奖金额。

活动背景图：移动端该抽奖活动的背景大图
中奖名单：控制是否于活动页面显示本次抽奖活动的中奖名单。
个人中奖记录：控制是否于活动页面显示本次抽奖活动中用户账号的中奖记录。
活动规则：开启后可添加抽奖活动规则说明。
活动状态：开启/关闭该活动，开启后移动端才会显示该活动

移动端展示

四、创建活动

1. 活动信息设置

（1）选择活动时间
（2）选择活动类型

  1. 积分抽奖：设置每次抽奖消耗的积分数额
  2. 订单支付：设置可抽奖的次数
  3. 商品评价：设置用户评价后可抽奖的次数
  4. 关注公众号：用户默认有一次抽奖次数，用户可邀请好友关注公众号获得抽奖次数，可设置获得抽奖次数的上限

2. 活动奖品设置

（1）默认有“未中奖“的设置
（2）奖品类型：
优惠券：选择的优惠券为赠送券
积分：中奖后自动发放到用户的积分账户
商品：选择的商品不能选择规格，需要在商品名称或活动规则中说明
红包：需开通商户自动到零钱功能才可使用
余额：中奖后自动发放到用户余额账户

3. 活动模块设置

关注公众号/积分抽奖，可设置活动页面的背景图、中奖名单、个人中奖记录、活动规则等，可在活动规则中展示此次抽奖活动的奖品信息、活动说明等
